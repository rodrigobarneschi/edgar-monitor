Great question! Let me be transparent about exactly what this agent does and its limitations.

What it actually does:

The agent polls (repeatedly asks) the EDGAR search API every 60 seconds. When it finds a filing with an ID it hasn't seen before in the current session, it flags it as "new" with a red badge counter.

So yes — you will be aware of new filings, BUT with some important caveats:

  1. It's not truly "real-time" — there's up to a 60-second delay between a filing being published and you seeing it. That's acceptable for most use cases since filings aren't time-critical to the second.
  2. The tab must stay open — this runs entirely in your browser. If you close the tab or your computer sleeps, monitoring stops. There's no background process, no server, no push notification.
  3. No sound/desktop alert — right now it only shows a red badge counter inside the app. You wouldn't notice a new filing if you're not looking at the tab.
  4. Session memory only — if you refresh the page, it forgets which filings it already saw and will re-flag everything as "new."

Great question! Let me explain the two approaches and their trade-offs:

Current approach: name-based full-text search

The monitor searches for the company name as a text string across all filing content. This actually works reasonably well for large Brazilian companies because:

  • EDGAR filings often mention the company name in multiple ways within the document
  • It catches filings about the company, not just by the company

But it has real weaknesses:

  • Vale could return false positives (the English word "vale" appears in many documents)
  • Bradesco in English filings might appear as "Banco Bradesco S.A." or just "Bradesco BBI"
  • It might miss filings if the company uses a slightly different registered name on EDGAR

CIK-based lookup: the precise way

Every company registered with the SEC has a unique CIK number (Central Index Key). Querying by CIK means:- You get every single filing from that entity, nothing more, nothing less

  • No false positives, no missed filings
  • The official registered name is used automatically

For example: Petrobras's CIK is 1119689. Querying https://data.sec.gov/submissions/CIK0001119689.json returns their complete, real-time filing history directly.

My recommendation

Use both together:

  1. CIK lookup as the primary source — precise, complete, zero false positives
  2. Name search as a fallback — for companies not yet registered on EDGAR or with uncertain CIKs
